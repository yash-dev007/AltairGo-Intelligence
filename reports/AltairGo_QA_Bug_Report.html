Bug Report

# Severity Component Bug Description Root Cause
1 🔴 HIGH Trip Planner Step 3 /recommend-destinations appears to timeout — UI stuck on “Scanning Satellite Data” with no error message shown to user Backend actually succeeds (~45 sec) but frontend fetch likely times out. Delay is caused by sequential Wikipedia image fetching for every AI-recommended destination.
2 🟡 MEDIUM Trip Planner Step 1 “Next Step” button stays disabled without clear feedback — users don’t know they must also set budget AND date before continuing No validation message or required-field indicator near the Next button.
3 🟡 MEDIUM Destination Cards Prices missing from destination cards and detail pages price_str field may be null/empty in the DB for seeded destinations.
4 🟡 MEDIUM Chat Widget Chat widget not found — no floating chat button in the DOM on any page ChatWidget component exists in /src/components/ChatWidget/ but may not be rendered in App.jsx or Home.jsx.
5 🟢 LOW Trip Planner Step indicators (1-5 circles at top) are non-clickable — users expect to click them to jump between steps Missing onClick handlers on step indicator elements.
6 🟢 LOW Destination Detail “Back to all destinations” button occasionally unresponsive (may require multiple clicks) Possible event handler issue or navigation guard.
7 🟢 LOW Destination Detail Reviews section always shows 0 reviews Review data not seeded in DB; reviews.json may not be loaded for all destinations.
8 🟢 LOW Trip Planner Country search input doesn’t auto-filter the grid effectively Search may only match exact names, or filtering logic has a debouncing issue.

Recommendations

1. Fix the /recommend-destinations timeout

Add a loading timeout on the frontend (60 seconds) with a user-friendly error message and retry button. Alternatively, move image fetching to a background job so the API returns quickly with placeholder images.

2. Add Chat Widget to layout

Verify ChatWidget is imported and rendered in App.jsx or the global layout component.

3. Seed price data

Populate price_str and estimated_cost_per_day during the migration script for all destinations.

4. Add validation feedback in Step 1

Show a tooltip or inline message near the “Next Step” button explaining that budget and travel dates must be set first.
